## Uncertainty-based Debiasing and Unlearning for Decontamination

### Model Contamination Process
```
bash train_contaminate.sh
```
You train 5 models with 5 different batch ordering, and save them in folders with `_order1` to `_order5`

Evaluate model performance
```
bash eval.sh
```
This will generate the result file defined by the `--outfile` argument

We provide our split for MMLU-pro and MATH-MCQA in `data/`
- D_dev: files end with `_true_dev.json`
- D_eval: files end with `_true_eval.json`

Due to space limitation, we will provide D_train later

### Decotamination
Get ensemble predictions from:
```
python correlation_batchorder.py
```
It requires result files once you run 5 contaminated models and 5 uncontaminated models. Please check details in the python script

To do uncertainty-based de-biasing
```
python check_distribution_quality.py
```
with
```
infile=../<your_contaminated_model_path>/<contaminated_result_path>.json
use_tilde=False
use_pred_alpha=True
shuffled=False
origfile=../<your_uncontaminated_model_path>/<uncontaminated_result_path>_ensemble.json
```
The origfile is generated by correlation_batchorder.py commented parts (see python script for details)

------

To run unlearning
```
bash train_unlearn.sh
```

Set:
```
traindata=results/ensemble_biased_model_truedeveval.json
```
which is the file containing ensemble debiased target for both dev and eval sets. Here are the steps to get it:
- You can get `results/ensemble_biased_model_trueeval.json` from `correlation_batchorder.py`
- You then change the file paths from contaminated to uncontaminated models to get `results/ensemble_biased_model_truedev.json`
- Finally you just concatenate both data together
